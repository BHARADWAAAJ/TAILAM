# Standard Implementation Note — IEC 60599

**Standard:** IEC 60599:2022, *Mineral oil-impregnated electrical equipment
in service — Guidance on the interpretation of dissolved and free gases
analysis.*
**Status in TAILAM:** Implemented. Frozen for Version 1.0.
**This document does not reproduce any text, table, or figure from the
standard.** It describes only how TAILAM's software applies it. Engineers
verifying TAILAM's output should refer to the official IEC 60599:2022
document, obtainable from IEC, for the authoritative thresholds and figures.

## Purpose

IEC 60599 is the primary international standard TAILAM implements for
interpreting dissolved gas concentrations in mineral oil. It covers both
the three-ratio diagnostic method and the Duval Triangle graphical method,
for both the main tank and (via its Annex A guidance) on-load tap changers.

## Scope of TAILAM's implementation

TAILAM implements four distinct pieces of IEC 60599 functionality:

1. **Duval Triangle 1** (main tank) — `src/js/engine/duval.js#calcDuval`.
   Classifies a sample into one of seven zones (PD, D1, D2, DT, T1, T2, T3)
   from the relative percentages of CH₄, C₂H₄ and C₂H₂, using the zone
   boundary geometry defined in the standard's main-tank triangle figure.
2. **Duval Triangle 2** (OLTC) — `src/js/engine/duval2.js#calcDuval2`.
   Classifies an OLTC oil-compartment sample into one of six zones (N, D1,
   X1, X3, T2, T3) from the same three gases, using the OLTC-specific
   triangle figure referenced in the standard's OLTC annex.
3. **Three-ratio method** — `src/js/engine/iec.js#calcIEC`. Computes the
   three diagnostic ratios and classifies the result into one of the
   standard's fault categories (PD, D1, D2, T1, T2, T3) or Indeterminate.
4. **The §9 "below-typical" gate** — `src/js/engine/duval2.js#applyBelowTypicalGate`.
   TAILAM implements the standard's guidance that an OLTC Duval Triangle 2
   fault zone should only be treated as an active fault when at least one
   gas exceeds its typical concentration; otherwise the classification is
   presented as an early pattern rather than a confirmed fault.

TAILAM also uses the standard's CO₂/CO ratio guidance (§5.5) for cellulose
(paper) involvement assessment — `iec.js#calcPaperInvolvement`. Dissolved-
oxygen interpretation is implemented separately under IEC 60422 guidance;
see `docs/standards/IEC60422.md` (the function lives in the same source
file, `iec.js`, for code co-location, but the interpretation bands
themselves are attributed to IEC 60422, not IEC 60599).

## How TAILAM implements it

Each method above is a pure function: it takes a gas-concentration object
in ppm and returns a result object (zone/fault code, name, description,
computed ratios or percentages). No user-editable configuration exists for
any threshold or zone boundary — they are fixed constants in the engine
source, reviewed against the standard once, and never modified outside a
dedicated, documented engineering-review sprint.

The Duval Triangle geometry is implemented as a point-in-polygon test
against the zone boundaries; the plotted triangles rendered in
`src/js/ui/charts.js` use the identical zone definitions the engine
classifies against, so the visual plot and the reported zone can never
disagree.

## Limitations

- TAILAM implements the standard's main-tank and OLTC interpretation
  guidance as encoded in its engine functions; it does not implement every
  ancillary table in the full standard (for example, sampling procedure
  guidance, which is outside the scope of a diagnostic calculator).
- TAILAM does not perform trend analysis across multiple samples in
  Version 1.0 — each analysis is evaluated as a single, independent
  sample. See `docs/release/ROADMAP.md`.
- Zone classification quality depends entirely on the accuracy of the
  gas concentrations entered; TAILAM performs no laboratory calibration or
  cross-lab correction.
